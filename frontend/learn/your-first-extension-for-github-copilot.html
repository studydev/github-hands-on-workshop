GitHub Copilot 확장이란?

Chat에 커스텀 기능을 추가하는 플러그인

Copilot 확장은 GitHub Chat과 github.com에 커스터마이징된 기능을 제공하는 추가 기능입니다.

확장으로 가능한 것

  • 문서 쿼리 — 서드파티 문서 서비스를 쿼리해 정보 검색
  • AI 지원 코딩 — 서드파티 AI 모델로 코드 제안 제공
  • 데이터 검색 — 외부 데이터 서비스에서 정보 조회
  • 작업 실행 — 메시지 게시판 포스팅, 외부 시스템 업데이트
완성된 확장은 GitHub Marketplace에 게시하여
전 세계와 공유할 수 있습니다!

확장의 3가지 구성 요소

GitHub App + Extension Service + Extension Content

1. GitHub App

메시지 처리 & Copilot과의 통신을 가능하게 하는 게이트웨이

2. Extension Service

메시지를 수신하고 커스텀 작업을 수행하는 웹 서비스 (Node.js)

3. Extension Content

확장에 제공되는 컨텍스트 자료 (마크다운 파일 등)

⚠️ Copilot 확장 ≠ VS Code의 Copilot 익스텐션
Copilot을 향상시키는 별도의 추가 기능입니다.

확장 서비스 실행하기

Step 2 — 템플릿 확장을 Codespace에서 실행

확장 서비스는 3개의 엔드포인트로 구성된 웹 서비스입니다.

필수 엔드포인트

  • GET /callback — 인증 후 사용자에게 제공하는 확인 메시지
  • POST /copilot — Copilot과 메시지를 교환하는 핵심 엔드포인트
  • GET / — (선택) 브라우저에서 확장 개요를 보여주는 정보 페이지

실행 방법

  • cd ghc-extension-js && npm install
  • 포트 3000을 public으로 설정
  • VS Code 디버거로 시작 → URL 확인
Codespace 재시작 시 포트 설정이 초기화됩니다.
반드시 가시성을 public으로 재설정하세요.

GitHub App 만들기

Step 3 — 확장을 GitHub에 연결하는 핵심 단계

GitHub App은 사용자에게 확장을 노출하고, 리소스에 대한 권한을 부여하는 중추입니다.

설정 흐름

  • Settings → Developer Settings → GitHub Apps
  • 앱 이름: my-ghc-extension-{사용자명}
  • Homepage URL: {Codespace URL}/info
  • Callback URL: {Codespace URL}/callback
  • Permissions: Copilot Chat → Read Only

Copilot 에이전트로 등록

  • App Type: Agent 선택
  • Agent URL: {Codespace URL}/copilot
  • Install App → 계정에 설치

컨텍스트로 커스터마이징

Step 4 — AI에게 맞춤 정보 주입하기

컨텍스트는 Copilot에게 제공하는 배경 정보입니다. system 역할의 메시지로 주입됩니다.

3가지 컨텍스트 파일

  • job-description.md — 확장 에이전트의 직무 설명
  • school-overview.md — 학교 이름·학기·학생 수 등 기본 정보
  • staff-roles.md — 교직원 역할별 업무 목록
마크다운 파일만 수정하면 코드 변경 없이
확장의 성격과 지식을 바꿀 수 있습니다.

index.js에서 메시지 주입

messages.unshift({ role: "system", content: ... }) 로 컨텍스트를 대화 시작 부분에 삽입합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: 브랜치 my-ghc-extension 생성 → create 이벤트 → 환경 확인
  • 2단계: 이슈 댓글 작성 → issue_comment 이벤트 → Codespace URL 확인
  • 3단계: 이슈 댓글 작성 → issue_comment 이벤트 → App 연결 확인
  • 4단계: 코드 커밋 → push 이벤트 → 컨텍스트 파일 확인
  • 5단계: PR 병합 → pull_request 이벤트 → 완료 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
