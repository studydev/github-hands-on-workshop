코드 보호 — Rulesets & .gitignore

더 많은 사람이 협업할수록 보호 조치가 필수입니다. GitHub는 두 가지 핵심 도구를 제공합니다.

1. 저장소 규칙 세트 (Repository Rulesets)

  • 중요한 브랜치에 직접 푸시 차단
  • 풀 리퀘스트 필수화 — 리뷰 없이 병합 불가
  • 브랜치 삭제 제한 & 강제 푸시 차단
  • Code Owners의 리뷰 필수 설정

2. .gitignore

  • 임시 파일, 비밀 설정, IDE 파일 등 추적 제외
  • 민감한 정보(.env) 실수로 커밋 방지
  • github/gitignore 템플릿 활용
규칙 세트를 설정하면 main에 직접 커밋할 수 없습니다!
모든 변경은 PR을 통해서만 반영됩니다.

협업 문서화 — 함께 일하는 규칙

CONTRIBUTING.md

새로운 기여자를 위한 "시작 가이드"입니다:

  • 개발 환경 설정 방법 (pip install -r requirements.txt)
  • 변경 사항 제안 과정 (브랜치 → PR → 리뷰)
  • 코딩 스타일 가이드 (PEP 8 등)
  • 도움을 요청하는 방법

CODEOWNERS

특정 파일·폴더의 책임자를 지정합니다:

  • 해당 파일 변경 시 자동으로 리뷰 요청
  • 핵심 코드(app.py, database.py)에 대한 관리 강화
PR을 생성하면 GitHub가 CODEOWNERS를 확인하고 자동으로 적절한 사람에게 리뷰를 요청합니다.

건강한 성장 — 커뮤니티 가이드라인

행동 강령 (Code of Conduct)

커뮤니티 구성원 간 상호작용 기대치를 설정합니다:

  • 환영하고 포용적인 언어 사용
  • 다양한 관점과 경험 존중
  • 건설적인 비평의 우아한 수용
  • 위반 시 보고 절차와 조치 명시

이슈 템플릿 (Issue Templates)

버그 보고나 기능 요청 시 표준화된 양식을 제공합니다:

  • Bug Report — 재현 단계, 예상 동작, 실제 동작
  • Feature Request — 문제 설명, 제안 솔루션
  • 충분한 정보로 효율적인 소통 가능
Contributor Covenant는 수많은 오픈소스 프로젝트에서 사용하는 인기 행동 강령입니다.

보안 대비 — 자동화된 취약점 관리

1. Dependabot

  • 의존성의 알려진 취약점을 자동으로 탐지
  • 안전한 버전으로 업그레이드 PR 자동 생성
  • 그룹화된 보안 업데이트로 PR 수 최소화

2. 코드 스캐닝 (CodeQL)

  • 저장소 코드의 보안 취약점 & 코딩 오류 분석
  • Copilot Autofix로 수정사항 자동 제안

3. 보안 정책 & 비공개 취약점 보고

  • SECURITY.md에 취약점 보고 절차 명시
  • Private vulnerability reporting으로 안전한 보고 채널 제공
  • 민감한 이슈가 수정 전 공개되는 것을 방지
보안은 선택이 아닌 필수입니다. GitHub의 도구들을 활용하면 최소한의 노력으로 대비할 수 있습니다.

릴리스 — PR 병합으로 완성

모든 준비가 끝나면 풀 리퀘스트를 병합하여 변경사항을 main에 반영합니다.

실습에서의 병합 흐름

main ──────────────────────────
  ├── prepare-to-collaborate ── ● merge ──▶
  └── add-issue-templates ── ● merge ──▶

병합 전 체크리스트

  • 보안 스캔이 완료되었는지 확인
  • CODEOWNERS의 리뷰 승인 확인
  • 규칙 세트의 모든 조건 충족 여부
두 개의 브랜치를 병렬로 작업하고 각각 병합하는 것이
실제 팀 협업의 모습입니다! 🎉

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • Step 1: Rulesets 설정 → push 이벤트 → .gitignore 존재 확인
  • Step 2: 파일 커밋 → push 이벤트 → CONTRIBUTING.md & CODEOWNERS 확인
  • Step 3: 템플릿 추가 → push 이벤트 → CODE_OF_CONDUCT.md 확인
  • Step 4: 보안 설정 → push 이벤트 → dependabot.yml & SECURITY.md 확인
  • Step 5: 병합 → pull_request closed → merged 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내를 남깁니다.
