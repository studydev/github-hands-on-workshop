OWASP Top 10

웹 애플리케이션 보안 위협 순위

OWASP(Open Worldwide Application Security Project)는 소프트웨어 보안 개선을 위한 비영리 재단으로, Top 10 목록은 가장 치명적인 웹 보안 리스크를 정리합니다.

주요 카테고리 (2021)

  • A01 — 접근 제어 실패 — 권한 없는 리소스 접근 허용
  • A02 — 암호화 실패 — 민감 데이터의 부적절한 보호
  • A03 — 인젝션 — SQL, LDAP, OS 명령 주입
  • A07 — 인증 실패 — 세션·비밀번호 취약점
Secure Code Game의 각 레벨은 OWASP Top 10 카테고리 중 하나 이상에 해당하는 취약점을 다룹니다.

인젝션 공격

신뢰할 수 없는 데이터가 명령어로 실행될 때

인젝션은 사용자 입력이 쿼리·명령의 일부로 해석되어 의도하지 않은 동작을 일으키는 공격입니다.

주요 유형

  • SQL Injection — ' OR 1=1 -- 로 DB 전체 읽기
  • LDAP Injection — 디렉터리 서비스 필터 조작
  • OS Command — ; rm -rf / 같은 시스템 명령 실행

방어 방법

  • 파라미터화 쿼리 — 입력값과 쿼리 구조 분리
  • ORM 사용 — 직접 SQL 작성 최소화
  • 입력 검증 — 허용 목록(allowlist) 기반 필터링
시즌 1 레벨 4(Data Bank)에서 SQL 인젝션을, 레벨 3(Social Network)에서 LDAP 인젝션을 직접 수정합니다.

인증 & 비밀번호 보안

사용자 신원 확인의 핵심 메커니즘

인증 실패는 비밀번호 저장·검증이 부적절할 때 발생합니다. 강력한 암호 정책만으로는 부족하며, 저장 방식이 핵심입니다.

취약한 패턴

  • 평문 저장 — DB 유출 시 전체 노출
  • 약한 해싱 — MD5, SHA-1은 레인보우 테이블 공격에 취약
  • 솔트 미사용 — 동일 비밀번호 → 동일 해시

안전한 방법

  • bcrypt / Argon2 — 느린 해싱 알고리즘 사용
  • 개별 솔트 — 사용자마다 고유한 랜덤 값 추가
  • MFA — 다중 인증 요소로 계층 방어
시즌 1 레벨 5(Locanda)에서 비밀번호 저장의 치명적 실수를 발견하고 수정합니다.

XSS (크로스 사이트 스크립팅)

악성 스크립트가 브라우저에서 실행될 때

XSS는 공격자가 웹 페이지에 악성 JavaScript를 삽입하여 다른 사용자의 브라우저에서 실행시키는 공격입니다.

XSS 유형

  • Stored XSS — 서버 DB에 스크립트 저장 → 모든 방문자 영향
  • Reflected XSS — URL 파라미터를 통한 즉시 반사
  • DOM-based XSS — 클라이언트 JavaScript에서 직접 조작

방어 방법

  • 출력 인코딩 — HTML 엔티티로 변환 (&lt;, &gt;)
  • CSP 헤더 — Content Security Policy로 스크립트 소스 제한
  • DOM 안전 API — textContent 사용, innerHTML 지양
시즌 1 레벨 3(Social Network)에서 사용자 입력이 템플릿에 주입되는 XSS 취약점을 수정합니다.

버퍼 오버플로우 & 메모리 안전

메모리 경계를 넘어서는 위험한 코드

버퍼 오버플로우는 프로그램이 할당된 메모리 영역을 초과하여 쓸 때 발생합니다. C/C++에서 가장 흔한 메모리 안전 취약점입니다.

공격 시나리오

  • 스택 오버플로우 — 리턴 주소 덮어쓰기로 코드 실행
  • 힙 오버플로우 — 동적 메모리 영역 조작
  • 정수 오버플로우 — 크기 계산 우회로 할당 크기 조작

방어 방법

  • 경계 검사 — 배열 인덱스·버퍼 크기 항상 확인
  • 안전한 함수 — strncpy 대신 strlcpy, snprintf
  • 메모리 안전 언어 — Rust, Go 같은 안전한 대안 채택
시즌 1 레벨 2(Matrix)에서 C 코드의 버퍼·정수 오버플로우를 직접 분석하고 수정합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions와 CodeQL을 사용해 여러분의 수정 사항을 자동으로 검증합니다.

동작 원리

  • 코드 수정 후 push → CodeQL 스캐닝 자동 실행
  • hack.py / tests.py 실행 → 보안 수정 검증
  • 모든 체크 통과 🟢 → 레벨 완료, 다음 레벨로 진행

시즌별 검증 방식

  • 시즌 1 — hack + tests 통과 + CodeQL 알림 해결
  • 시즌 2 — Actions 워크플로우 자체가 레벨 소재
  • 시즌 3 — AI 관련 보안 검증 (프롬프트 인젝션 등)
Codespaces에서 실행하면 모든 도구가 사전 구성되어 있으므로 즉시 시작할 수 있습니다.
