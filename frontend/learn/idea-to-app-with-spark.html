GitHub Spark란?

코딩 없이 대화로 앱을 만드는 AI 도구

GitHub Spark는 비개발자를 위해 설계된 도구입니다. 소프트웨어 용어 없이 일상 언어로 설명하면 실제 웹 애플리케이션을 만들어 줍니다.

새 동료에게 설명하듯이 아이디어를 전달하면
Spark가 몇 분 안에 앱으로 구현합니다.

Spark의 핵심 특징

  • 소프트웨어 용어 없음 — 대화하듯 협업
  • 실시간 미리보기 — 변경 사항 즉시 확인
  • 관리할 시스템 없음 — 인프라 걱정 불필요
  • 빠름 — 프로토타입까지 며칠이 아닌 몇 분

과금 모델

프리미엄 요청 단위(PRU) 사용 — 각 요청은 4 PRU를 소비합니다. 아이디어를 긴 하나의 요청으로 결합하면 효율적!

좋은 설명 작성하기

명확한 소통이 좋은 결과의 핵심

Spark는 질문을 피하고 창의적으로 자유롭게 판단하는 경향이 있습니다. 원하는 결과를 얻으려면 명확하게 설명하세요.

예제 앱 시나리오

  • ☕️ 커피숍 시크릿 메뉴 — 짧은 프롬프트
  • 🥍 라크로스 통계 트래커 — 상세한 요구사항
  • 🍽️ 종이 접시 디자이너 — 기술적 명세 + 이미지 첨부
같은 설명으로 여러 앱을 시작하면 다른 결과를 얻습니다.
병렬 아이디어를 탐색하는 좋은 방법!

팁

GitHub.com의 Copilot 같은 AI 채팅 도구로 아이디어를 브레인스토밍한 뒤 Spark에 전달하면 더 효과적입니다.

지속적인 개선

Step 2 — 대화로 앱을 계속 발전시키기

6개월간 기다릴 필요 없이 첫날부터 몇 번의 대화만으로 유용한 앱을 가질 수 있습니다!

아이디어 전달 방법

  • 왼쪽 채팅 패널 — 원하는 변경 사항을 자유롭게 설명
  • 요소 선택기 — 미리보기에서 특정 영역을 클릭 & 수정 요청
  • 이미지 첨부 — 목업이나 손 스케치로 설명을 명확히
실수했나요? 되돌리기(Restore)로 이전 버전으로 돌아간 뒤
더 구체적인 설명으로 다시 시도하세요!

탐색할 상황들

  • 기존 기능 개선 · 새 기능 추가
  • 버그 수정 · 불필요한 기능 제거

비주얼 & 데이터 커스터마이징

Step 3 — 테마, 에셋, 데이터 관리

코딩을 몰라도 되듯이, 색상 전문가가 될 필요도 없습니다. Spark가 미리 만든 테마를 제공합니다.

커스터마이징 도구

  • Theme 탭 — 색상 테마, 타이포그래피, 간격 조정
  • Color 영역 — 테마 오버라이드로 수동 색상 설정
  • Assets 탭 — 로고·이미지 업로드 & 통합
  • Data 탭 — 키-값 저장소 확인 & 수동 편집
Spark가 앱의 데이터를 위한 키-값 저장소를 자동 제공하며
내장 인터페이스로 저장된 정보를 관리할 수 있습니다.

앱 공유 & 고급 개발자 기능

Step 4–5 — 게시, Codespace, 저장소

공개 수준 (3단계)

  • 나만 (Private) — 내 계정만 접근
  • 조직 (Organization) — 조직 구성원과 공유
  • 모든 GitHub 사용자 — 로그인한 누구나 접근

데이터 공유

  • 읽기 전용 — 보고 사용만 가능 (권장)
  • 쓰기 접근 — 데이터 수정 가능 (소규모 협업용)
모든 Spark 앱 뒤에는 GitHub Codespace가 있습니다.
코드를 직접 편집하고 저장소로 내보내기도 가능!

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: 예제 선택 체크박스 → issue 이벤트 → 앱 생성 확인
  • 2단계: Spark 기능 선택 → issue 이벤트 → 수정 완료 확인
  • 3단계: 스타일 선택 → issue 이벤트 → 디자인 확인
  • 4단계: 공유 선택 → issue 이벤트 → 게시 확인
  • 5단계: (선택) 고급 기능 → issue 이벤트 → 완료 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
