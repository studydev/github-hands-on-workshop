VS Code 로컬 코드 리뷰

GitHub Copilot은 VS Code에서 직접 코드를 리뷰하여 커밋 전 즉각적인 피드백을 제공합니다.

주요 기능

  • 커밋하지 않은 변경사항의 로컬 분석
  • 코드 품질 및 스타일 권장사항
  • 일반적인 보안 취약점 탐지
  • 성능 최적화 제안
소스 컨트롤 패널 → Code Review 버튼 클릭 →
Comments 패널에서 피드백 확인 → Apply / Discard

리뷰 레벨

unstaged staged uncommitted

Pull Request 코드 리뷰

병합 전 Copilot의 자동 코드 분석

Copilot은 코드를 분석하고 즉시 적용할 수 있는 제안과 함께 지능적인 피드백을 제공합니다.

주요 특징

  • 자동 분석 — 품질, 보안, 성능 문제 리뷰
  • 실행 가능한 제안 — 코드 변경과 함께 권장사항
  • PR 플로우 통합 — GitHub 네이티브 리뷰와 동일
  • 비차단 — Comment 리뷰로 병합을 차단하지 않음
  • 커스터마이징 — 팀 표준에 맞춘 지침 지원
각 코드 리뷰 = 1 PRU (Premium Request Unit) 소모
Reviewers에서 Copilot을 선택하여 리뷰 요청

저장소 커스텀 지침

Copilot에 프로젝트 표준과 선호사항 전달

지침 유형

  • 저장소 전체 지침
    .github/copilot-instructions.md
  • 경로별 지침
    .github/instructions/*.instructions.md

경로별 지침 예시

---
applyTo: "*.html,*.css,*.js"
---
## Frontend Guidelines …
자연어 + Markdown으로 보안, 코드 표준,
성능, 스타일 가이드 등을 정의합니다.

자동 리뷰를 위한 규칙 세트

모든 PR에 자동 코드 리뷰 강제하기

저장소 규칙 세트로 모든 Pull Request에 자동 리뷰를 강제하여 일관된 품질을 보장합니다.

설정 방법

Settings → Rules → Rulesets → New branch ruleset

주요 규칙

  • ☑️ Require a pull request before merging
  • ☑️ Require conversation resolution
  • ☑️ Automatically request Copilot code review
저장소·브랜치·조직 수준으로 적용 가능
브랜치 보호와 기존 워크플로우에 자연스럽게 통합

코드 리뷰 완전 워크플로우

로컬에서 자동화까지 — 전체 흐름 한눈에

코드 작성 → 로컬 리뷰 → 커밋 & 푸시 → PR 리뷰 → 병합

달성하는 것

  • VS Code 로컬 리뷰 — 커밋 전 즉각 피드백
  • PR Copilot 리뷰 — 병합 전 자동 분석
  • 커스텀 지침 — 팀 고유 표준 적용
  • 규칙 세트 자동화 — 수동 개입 없이 일관된 품질
개발 과정 초기부터 끝까지
AI가 코드 품질을 보장하는 완전한 파이프라인

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: 브랜치 푸시 → push 이벤트 감지 → 로컬 리뷰 완료 확인
  • 2단계: PR 생성 → pull_request 감지 → Copilot 리뷰 요청 확인
  • 3단계: 지침 커밋 → push 이벤트 감지 → 커스텀 지침 파일 확인
  • 4단계: PR 병합 → pull_request closed 감지 → 규칙 세트 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
