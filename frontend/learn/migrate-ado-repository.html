GitHub Enterprise Importer

Azure DevOps → GitHub 공식 마이그레이션 도구

GitHub Enterprise Importer (GEI)는 Azure DevOps에서 GitHub Enterprise Cloud로 높은 충실도의 마이그레이션을 제공하는 공식 도구입니다.

왜 GEI를 사용하나요?

  • Git 히스토리 100% 보존 — 모든 커밋, 브랜치, 태그 유지
  • PR 메타데이터 이전 — 리뷰, 댓글, 작업 항목 링크까지
  • 브랜치 정책 변환 — ADO 보호 규칙을 GitHub 규칙으로
  • CLI 기반 — 스크립트로 대규모 일괄 마이그레이션 가능
git clone --mirror보다 훨씬 많은 데이터를 안전하게 옮깁니다.

Personal Access Token 인증

양쪽 플랫폼을 안전하게 연결하는 열쇠

마이그레이션에는 두 개의 PAT이 필요합니다 — Azure DevOps(소스)와 GitHub(대상).

Azure DevOps PAT 범위

  • Project and Team — Read, write & manage
  • Code — Read, write & manage
  • Work Items — Read & write
  • Build — Read & execute
  • Identity — Read

GitHub PAT 범위

  • repo, admin:org, workflow
  • 조직 소유자(Owner) 권한 필요
PAT은 실습 후 반드시 삭제하세요 — 만료 기간을 1일로 설정하는 것을 권장합니다.

ado2gh CLI 확장

GitHub CLI에 마이그레이션 능력을 더하다

gh ado2gh는 GitHub CLI의 확장(extension)으로, ADO 저장소를 GitHub으로 마이그레이션하는 명령어를 제공합니다.

설치 및 사용

# CLI 확장 설치
$ gh extension install github/gh-ado2gh

# 저장소 마이그레이션 실행
$ gh ado2gh migrate-repo \
  --ado-org my-org \
  --ado-team-project my-project \
  --ado-repo my-repo \
  --github-org my-gh-org \
  --github-repo migrated-repo

핵심 명령어

  • grant-migrator-role — migrator 역할 부여
  • migrate-repo — 저장소 마이그레이션 실행
  • revoke-migrator-role — migrator 역할 회수

마이그레이션되는 데이터

하나도 안 버리고 통째로 옮기기

GitHub Enterprise Importer는 다음 데이터의 마이그레이션을 지원합니다:

  • Git 소스 — 커밋 히스토리 전체
  • Pull Request — 리뷰, 댓글, 상태
  • 사용자 히스토리 — PR 작성자, 리뷰어 매핑
  • 작업 항목 링크 — AB#<id> 참조 보존
  • 첨부 파일 — PR에 붙은 이미지·문서
  • 브랜치 정책 — 보호 규칙 변환
마이그레이션 후 자동 생성되는 마이그레이션 로그 Issue에서 경고·오류를 확인하세요.

Azure DevOps ↔ GitHub 통합

두 플랫폼의 장점을 동시에

마이그레이션 후에도 Azure Boards와 Azure Pipelines를 GitHub 저장소에 연동하여 하이브리드 워크플로우를 구성할 수 있습니다.

Azure Boards 통합

  • 커밋·PR에 AB#<id>로 작업 항목 자동 연결
  • fixes AB#<id>로 머지 시 작업 항목 자동 완료
  • 작업 항목에서 직접 GitHub 브랜치 생성

Azure Pipelines 통합

  • GitHub 저장소 변경으로 Azure Pipelines 자동 트리거
  • PR에서 빌드 상태 직접 확인
GitHub의 AI 기능 + Azure DevOps의 프로젝트 관리·CI/CD — 최고의 조합!

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: ADO 환경 설정 → bootstrap 스크립트 완료 감지
  • 2단계: 마이그레이션 도구 설정 → config.yml push 감지
  • 3단계: 저장소 마이그레이션 → issue_comment 감지 → 마이그레이션 완료 확인
  • 4단계: ADO 통합 → issue_comment 감지 → 통합 테스트 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
