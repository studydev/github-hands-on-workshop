마크다운 (Markdown)

개발자의 글쓰기 도구 — 가볍고 강력한 텍스트 서식 언어

마크다운은 일반 텍스트에 간단한 기호를 더해 서식 있는 문서를 만드는 경량 마크업 언어입니다.

GitHub에서는 거의 모든 곳에서 마크다운을 사용합니다:

  • README.md — 프로젝트 소개 문서
  • Issue & PR — 버그 리포트, 코드 리뷰 코멘트
  • .md 파일 — 문서, 가이드, 위키
  • GitHub Pages — 정적 웹사이트 콘텐츠
마크다운을 배우면 GitHub 어디서든 체계적이고 보기 좋은 글을 쓸 수 있습니다.

헤딩 (Heading)

문서의 뼈대를 세우는 제목 구조

헤딩은 # 기호를 사용해 문서의 계층 구조를 만듭니다. 1~6단계까지 지원됩니다.

# 제목 1 — 가장 큰 제목 (H1)
## 제목 2 — 섹션 제목 (H2)
### 제목 3 — 하위 섹션 (H3)
#### 제목 4 — 더 세부 항목 (H4)

잘 쓰는 팁

  • H1은 문서 당 하나만 사용 (문서 제목)
  • H2로 큰 섹션, H3로 세부 항목을 나눔
  • GitHub은 헤딩으로 자동 목차를 생성
실습 1단계에서 index.md 파일에 헤딩을 추가합니다.

순서 없는 목록:

- 항목 2

마크다운 서식은 Issue, PR 코멘트, README 어디서든 동일하게 작동합니다.

작업 목록 (Task List)

체크박스로 할 일을 추적하는 체크리스트

작업 목록은 GitHub에서 Issue와 PR에 특히 유용한 기능입니다.

- [x] 완료된 항목
- [ ] 아직 안 한 항목
- [ ] 다음에 할 일

GitHub에서의 특별한 기능

  • Issue에 작업 목록을 넣으면 진행률 바가 표시됨
  • 체크박스를 클릭만으로 토글 가능 (편집 불필요)
  • PR 리뷰에서 리뷰 체크리스트로 활용
실습 2단계에서 index.md에 작업 목록을 추가합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: 헤딩 추가 → push 이벤트 감지 → # 문법 확인
  • 2단계: 작업 목록 → push 이벤트 감지 → - [ ] 문법 확인
  • 3단계: 코드 블록 → push 이벤트 감지 → ``` 문법 확인
  • 4단계: 이미지 추가 → push 이벤트 감지 → ![]() 문법 확인
  • 5단계: PR 병합 → pull_request closed 감지
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
