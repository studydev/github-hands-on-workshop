GitHub Packages & GHCR

코드와 함께 패키지를 관리하는 플랫폼

GitHub Packages는 컨테이너와 의존성을 호스팅하고 관리하는 플랫폼입니다. Docker, npm, Maven, NuGet 등 다양한 레지스트리를 지원합니다.

Container Registry (ghcr.io)

  • ghcr.io — GitHub의 Docker 이미지 전용 레지스트리
  • GITHUB_TOKEN으로 자동 인증 — 별도 시크릿 불필요
  • packages: write 권한만 설정하면 즉시 사용
  • 공개 저장소에서 무료
패키지는 저장소가 아닌 계정(사용자/조직)에 연결됩니다.
여러 저장소에서 동일한 패키지를 재사용할 수 있습니다.

Dockerfile & 기본 빌드

애플리케이션을 컨테이너로 패키징하는 첫 걸음

Dockerfile은 Docker 이미지를 만드는 레시피입니다. 워크플로우에서 docker build로 이미지를 만들고 docker push로 레지스트리에 게시합니다.

기본 워크플로우 구조

name: Docker Publish
on:
  push:
    branches: [main]
permissions:
  contents: read
  packages: write
jobs:
  build-and-push:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: docker/login-action@v3
      - run: docker build & push
docker/login-action으로 ghcr.io에 인증하고
GITHUB_TOKEN을 비밀번호로 사용합니다.

전문 Docker Actions

커뮤니티가 만든 고급 빌드 도구

직접 docker 명령어 대신 공식 Docker Actions를 사용하면 멀티 플랫폼 빌드, 캐싱, 고급 기능을 쉽게 활용할 수 있습니다.

핵심 Docker Actions

  • docker/login-action — ghcr.io 인증
  • docker/setup-qemu-action — ARM64 등 다른 아키텍처 에뮬레이션
  • docker/setup-buildx-action — BuildKit 기반 고급 빌드 엔진
  • docker/build-push-action — 멀티 플랫폼 빌드 & 푸시
- uses: docker/setup-qemu-action@v3
- uses: docker/setup-buildx-action@v3
- uses: docker/build-push-action@v6
  with:
    platforms: linux/amd64,linux/arm64

메타데이터 & 동적 태깅

Git 컨텍스트에 따라 자동으로 태그 생성

docker/metadata-action은 워크플로우를 트리거한 Git 컨텍스트(브랜치, 태그, PR)에 따라 Docker 태그를 자동 생성합니다.

이벤트별 태그 매핑

  • 브랜치 푸시 → main
  • Pull Request → pr-2
  • 태그 푸시 → v1.2.3, latest
  • workflow_dispatch → main
- name: Extract metadata
  id: meta
  uses: docker/metadata-action@v5
  with:
    images: ghcr.io/OWNER/APP
- uses: docker/build-push-action@v6
  with:
    tags: ${{ steps.meta.outputs.tags }}

라이프사이클 기반 게시

개발 사이클에 맞춘 자동 버전 관리

실제 소프트웨어 개발에서는 다양한 단계에서 Docker 이미지를 게시합니다. 다중 트리거로 전체 라이프사이클을 자동화할 수 있습니다.

트리거 & 태깅 전략

  • 새 커밋 (push) — 브랜치명 + SHA 태그
  • Pull Request — pr-N 태그로 테스트 빌드
  • 버전 릴리스 (tag) — v1.0.0 + latest
  • 수동 실행 — workflow_dispatch 지원
on:
  push:
    branches: [main]
    tags: ["v*"]
  pull_request:
    branches: [main]
  workflow_dispatch:
하나의 워크플로우로 개발 → 테스트 → 릴리스
전체 파이프라인을 자동화할 수 있습니다.

실습의 비밀 — 자동 검증

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

자동 검증 흐름

  • 1단계: 기본 워크플로우 생성 → push 이벤트 → YAML 구조 확인
  • 2단계: Docker Actions 적용 → push 이벤트 → 멀티 플랫폼 빌드 확인
  • 3단계: 메타데이터 액션 추가 → push 이벤트 → 동적 태깅 확인
  • 4단계: 릴리스 생성 → tag 이벤트 → 버전 태그 이미지 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내를 남깁니다.
