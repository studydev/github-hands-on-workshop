레이블 기반 트리거

PR 레이블로 워크플로우를 선택적으로 실행

레이블을 풀 리퀘스트에 붙이면 GitHub Actions가 해당 레이블에 맞는 작업만 실행합니다. 하나의 워크플로우에서 여러 시나리오를 관리할 수 있습니다.

이 실습의 레이블

  • spin up environment — Azure 리소스 생성
  • stage — 스테이징 환경에 배포
  • destroy environment — 리소스 정리
on:
  pull_request:
    types: [labeled]

jobs:
  build:
    if: contains(
      github.event.pull_request
        .labels.*.name, 'stage')
레이블 하나로 배포 파이프라인을 제어하는 것이 핵심입니다.

Azure 리소스 구성

GitHub Actions에서 Azure CLI로 인프라 생성

워크플로우에서 Azure CLI를 사용해 리소스 그룹, App Service 플랜, 웹 앱을 자동으로 생성합니다.

생성하는 리소스

  • 리소스 그룹 — 관련 리소스를 묶는 컨테이너
  • App Service 플랜 — 웹 앱 호스팅 및 요금 관리 (F1 무료)
  • 웹 앱 — 컨테이너 이미지를 실행하는 서비스

필요한 시크릿

  • AZURE_CREDENTIALS — 서비스 주체 JSON
  • AZURE_SUBSCRIPTION_ID — 구독 ID
  • CR_PAT — 컨테이너 레지스트리 토큰
azure/login 액션으로 인증 후 az CLI 명령을 실행합니다.

Docker 이미지 빌드 & 푸시

GitHub Container Registry(GHCR)에 이미지 저장

애플리케이션을 Docker 이미지로 빌드하고 GHCR에 푸시합니다. Azure 웹 앱이 이 이미지를 풀하여 실행합니다.

빌드 파이프라인

  • actions/checkout — 소스 코드 체크아웃
  • actions/setup-node — Node.js 환경 설정
  • docker/login-action — GHCR 로그인
  • docker/build-push-action — 이미지 빌드 & 푸시
- uses: docker/login-action@v3
  with:
    registry: ghcr.io
    username: ${{ github.actor }}
    password: ${{ secrets.CR_PAT }}

- uses: docker/build-push-action@v5
  with:
    push: true
    tags: ${{ steps.meta.outputs.tags }}

스테이징 환경 배포

PR 레이블로 스테이징에 먼저 배포

풀 리퀘스트에 stage 레이블을 붙이면 스테이징 배포 워크플로우가 실행됩니다. 프로덕션 전에 안전하게 검증합니다.

deploy-staging.yml 흐름

  • build — npm install & webpack 빌드
  • Build-Docker-Image — 이미지 빌드 후 GHCR 푸시
  • Deploy-to-Azure — azure/webapps-deploy로 배포
- uses: azure/webapps-deploy@v3
  with:
    app-name: ${{ env.AZURE_WEBAPP_NAME }}
    images: ghcr.io/.../${{ github.sha }}
스테이징에서 게임을 직접 플레이하여 배포가 제대로 되었는지 확인하세요!

프로덕션 배포 & 정리

main 브랜치 병합 시 자동으로 프로덕션 배포

프로덕션 워크플로우는 main 브랜치에 푸시될 때 실행됩니다. 스테이징과 동일한 파이프라인이지만 트리거가 다릅니다.

스테이징 vs 프로덕션

  • 스테이징 — pull_request: [labeled] + stage 레이블
  • 프로덕션 — push: branches: [main]

환경 정리

  • destroy environment 레이블 → 리소스 그룹 삭제
  • 무료 시간 소진 방지를 위해 반드시 정리
on:
  push:
    branches:
      - main
프로덕션 배포 후 반드시 destroy environment 레이블로 리소스를 정리하세요!

사용하는 주요 Actions

이 실습에서 활용하는 오픈소스 Actions

빌드 & 아티팩트

  • actions/checkout@v4 — 코드 체크아웃
  • actions/setup-node@v4 — Node.js 설정
  • actions/upload-artifact@v4 — 빌드 결과 업로드
  • actions/download-artifact@v4 — 아티팩트 다운로드

Docker & Azure

  • docker/login-action@v3 — 레지스트리 인증
  • docker/build-push-action@v5 — 이미지 빌드·푸시
  • azure/login@v2 — Azure 인증
  • azure/webapps-deploy@v3 — 웹 앱 배포
바퀴를 다시 발명하지 마세요!
