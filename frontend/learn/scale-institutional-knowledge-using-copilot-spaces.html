Copilot Spaces란?

AI 지식 허브 — 흩어진 정보를 한 곳에

Copilot Spaces는 리포지토리, 코드, PR, 이슈, 메모 등을 한 곳에 모아 Copilot이 더 정확하게 답변할 수 있게 만드는 협업 환경입니다.

팀의 암묵적 지식을 검색 가능하고 버전 관리되는 문서로 전환하여
특정 개인 의존도를 줄이고 온보딩을 가속화합니다.

Spaces로 할 수 있는 것

  • Copilot으로부터 더 관련성 높은 답변 확보
  • 팀과 지식 공유하여 반복 질문 감소
  • GitHub 파일 변경 시 자동 동기화
  • 채팅 기록을 넘어 지속되는 컨텍스트 제공

지식 허브 설정 — Step 1

Space 생성 → 지침 추가 → 리포 연결 → 이슈 생성

OctoAcme 프로젝트 관리 허브를 만들고, Copilot이 문서를 이해할 수 있도록 지침과 소스를 설정합니다.

설정 순서

  • Space 생성 — github.com/copilot/spaces에서 생성
  • 지침 추가 — 목적과 문서 구조를 Markdown으로 정의
  • 리포 소스 연결 — docs/와 .github/ISSUE_TEMPLATE 추가
  • 이슈 생성 — README 작성 이슈를 Copilot에게 요청
지침은 Copilot이 어떤 문서가 어디에 있는지,
Space의 목적이 무엇인지 이해하게 돕습니다.

문서 탐색 & 요약 — Step 2

프로세스 문서를 읽고, 요약하고, PR로 만들기

Copilot Space에서 프로젝트 관리 프로세스 문서를 탐색하고 3-4 문단으로 요약한 뒤, 코딩 에이전트로 README를 자동 생성합니다.

요약 시 주목할 포인트

  • 엔드투엔드 워크플로우 단계와 핸드오프
  • 정의된 역할/페르소나와 책임
  • 의사결정 / 에스컬레이션 포인트
  • 커뮤니케이션 주기와 채널
  • 품질 게이트, 리뷰, 인수 기준
이슈를 대화에 첨부한 뒤
github-coding-agent로 PR을 자동 생성합니다.

코딩 에이전트 — 이슈에서 PR까지

Copilot이 브랜치 생성 → 코드 변경 → PR 생성까지

Copilot Space 대화에서 이슈를 첨부하고, github-coding-agent 도구로 자동 PR을 생성합니다.

동작 흐름

  • 이슈 URL을 @repo/issues/# 형태로 첨부
  • 프롬프트로 PR 생성 지시 → Allow 클릭
  • Agent Sessions에서 진행 상황 모니터링
  • PR 완료 후 리뷰 & 머지
⚠️ 코딩 에이전트는 작업 완료에 5-15분 소요됩니다.
View session으로 실시간 진행 확인이 가능합니다.

반복적 프로세스 개선 — Step 3

갭 식별 → 이슈 → 구현 → 머지 — 완전한 개선 주기

이슈 템플릿을 활용해 프로세스의 부족한 부분을 식별하고, 개선 사항을 이슈로 기록한 뒤, 코딩 에이전트로 구현 PR을 생성합니다.

개선 워크플로우

  • 갭 식별 — 누락된 페르소나/역할, 모호한 프로세스
  • 이슈 생성 — 이슈 템플릿으로 구조화된 이슈 작성
  • PR 생성 — 코딩 에이전트가 docs/ 업데이트
  • 리뷰 & 머지 — Approve → Merge 완료
이 패턴은 프로젝트 관리를 넘어
지식을 캡처·공유해야 하는 모든 분야에 적용 가능합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: issues 이벤트 → 이슈 생성 여부 확인
  • 2단계: pull_request 이벤트 → README PR 확인
  • 3단계: pull_request 이벤트 → 개선 PR 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
