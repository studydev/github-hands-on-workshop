GitHub Flow & 릴리스

지속적 배포와 버전 릴리스의 차이

GitHub flow는 정기적으로 배포하는 프로젝트를 위한 가벼운 브랜치 기반 워크플로우입니다.

두 가지 배포 모델

  • 지속적 배포 (CD) — main에 커밋할 때마다 자동 릴리스
  • 릴리스 기반 — 버전과 릴리스를 위해 다른 구조에 의존
릴리스 기반 워크플로우는 GitHub flow 위에
태그, 릴리스, 릴리스 브랜치를 추가합니다.

버전이란?

업데이트된 소프트웨어의 다양한 반복입니다. 개발자들은 코드 업데이트 후 테스트를 실행하고, 보안 설정을 한 뒤, 테스트된 코드를 릴리스합니다.

릴리스 관리 전략

브랜치 전략으로 릴리스를 체계적으로

릴리스를 관리하는 전략은 다양합니다. 이 실습에서는 release-v1.0 브랜치를 릴리스 버전당 하나의 장기 브랜치로 사용합니다.

릴리스 브랜치 전략

  • 장기 브랜치 — production, dev, main
  • 기능 브랜치 — 단순한 수명 짧은 브랜치로 릴리스
  • 릴리스 브랜치 — 버전별 release-v1.0 패턴

보호된 브랜치

릴리스 브랜치는 main과 마찬가지로 브랜치 보호 규칙을 설정할 수 있습니다. 강제 푸시나 실수로 인한 삭제를 방지합니다.

가능한 한 장기 브랜치를 줄이고,
각 브랜치에 항상 의도적인 목적을 부여하세요.

시맨틱 버전닝

MAJOR.MINOR.PATCH — 호환성을 명확히 전달

시맨틱 버전닝은 세 부분으로 구성된 버전 번호를 사용하여 호환성을 지정하는 공식 규약입니다.

버전 규칙

코드 상태단계예시
첫 번째 릴리스새 제품1.0.0
하위 호환 수정패치1.0.1
하위 호환 기능 추가마이너1.1.0
호환성 깨는 변경메이저2.0.0
이 실습에서는 v0.9(베타) → v1.0.0(릴리스) → v1.0.1(핫픽스)
의 버전 흐름을 직접 경험합니다.

릴리스 노트 자동 생성

변경사항을 자동으로 문서화

자동 생성 릴리스 노트는 릴리스 내용의 개요를 빠르게 만들어줍니다.

자동 생성되는 내용

  • 머지된 PR 목록 — 변경사항 요약
  • 기여자 목록 — 릴리스 참여자
  • 전체 변경 로그 링크 — 커밋 비교

릴리스 워크플로우

1. Releases 페이지 → Draft a new release
2. 태그 버전 입력 (예: v1.0.0)
3. Target 브랜치 선택
4. Generate release notes 클릭
5. 내용 검토 및 커스터마이즈
6. Publish release 또는 Save draft
릴리스 노트 생성 후 사용자 지정도 가능합니다.
명확한 PR 설명이 좋은 릴리스 노트의 핵심입니다.

핫픽스 전략

프로덕션 버그를 빠르게 수정하고 배포

핫픽스(Hotfix)는 릴리스 후 발견된 버그를 빠르게 수정하기 위한 긴급 패치입니다.

핫픽스 워크플로우

  • 핫픽스 브랜치 생성 — hotfix-v1.0.1 브랜치에서 작업
  • 버그 수정 — fix-game-background 같은 수정 브랜치
  • 핫픽스에 머지 — 수정 PR 생성 후 핫픽스 브랜치에 머지
  • main에 머지 — 핫픽스 → main PR 생성 후 머지
  • 패치 릴리스 — v1.0.1 태그로 새 릴리스 생성
핫픽스 후에도 브랜치를 삭제하지 않습니다.
같은 브랜치에서 추가 수정이 필요할 수 있기 때문입니다.

실습의 비밀 — 자동 검증

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

자동 검증 흐름

  • Step 1: 릴리스 생성 → release 이벤트 → v0.9 베타 확인
  • Step 2: 기능 PR 머지 → pull_request closed → 릴리스 브랜치 업데이트
  • Step 3: 릴리스 PR 열기 → pull_request opened → base:main 확인
  • Step 4: 릴리스 노트 생성 → release drafted → 노트 확인
  • Step 5: 릴리스 마무리 → release published → v1.0.0 게시
  • Step 6: 핫픽스 커밋 → pull_request closed → main에 머지
  • Step 7: 핫픽스 릴리스 → release published → v1.0.1 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내를 남깁니다.
