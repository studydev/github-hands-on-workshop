풀 리퀘스트 (Pull Request)

브랜치 작업을 리뷰하고 다듬는 협업 공간

풀 리퀘스트는 한 브랜치의 작업을 다른 브랜치에 병합하기 전에 리뷰하는 공간입니다.

PR의 4가지 탭

  • Conversation — 활동 로그, 피드백과 토론의 장
  • Commits — 제안된 브랜치의 커밋 목록
  • Checks — GitHub Actions 자동화 결과
  • Files Changed — Diff 뷰 + 라인별 댓글
실습 1단계에서 update-game → main 방향으로 PR을 만듭니다.

담당자 (Assignee)

이 변경 사항에 대해 누구에게 물어봐야 하나요?

PR 담당자는 제안된 변경 사항에 가장 익숙한 사람입니다. 질문이 있을 때 누구에게 연락해야 하는지 추적하는 메커니즘입니다.

담당자 vs 리뷰어

  • 담당자 (Assignee) — PR의 책임자, 변경 관련 질문 대상
  • 리뷰어 (Reviewer) — 코드를 검토하고 피드백 제공
  • 한 PR에 여러 명의 담당자와 리뷰어 지정 가능
PR 생성 ──→ Assignee 지정 ──→ 리뷰 요청
실습 2단계에서 오른쪽 사이드바의 assign yourself를 클릭해 본인을 담당자로 지정합니다.

코드 리뷰 — 3가지 피드백

Comment, Approve, Request Changes

리뷰어가 PR에 피드백을 제공하는 3가지 옵션입니다:

Comment | Approve | Request Changes
  • Comment — 승인이나 거부 없는 일반적인 피드백
  • Approve — 병합을 허용하는 승인 (규칙 세트/CODEOWNERS 연동)
  • Request Changes — 추가 작업이 필요함, 변경 후 재검토 요청
실습 3단계에서 Files changed 탭에서 코드를 검토하고 Comment 리뷰를 제출합니다.

리뷰는 어떻게 하나요?

좋은 리뷰를 위한 프로세스와 마음가짐

리뷰 프로세스

  • 1. 제목과 설명이 명확한지 검토
  • 2. Files changed에서 코드가 설명과 일치하는지 확인
  • 3. 가능하면 직접 실행해 보기
  • 4. 기여 가이드 기준 충족 확인

리뷰 베스트 프랙티스

  • 잠재적 문제, 위험, 제한 사항을 식별
  • 중요한 피드백에 우선순위를 매기기
  • 잘한 점을 칭찬하기
  • 저자를 친절하고 공감적으로 대하기

변경 제안 (Suggestion)

설명보다 직접 코드를 수정 제안하기

오타나 간단한 수정은 설명보다 직접 제안하는 것이 효율적입니다. Add a suggestion 기능으로 가능합니다.

제안 워크플로우

  • 1. Files changed에서 수정할 줄 옆 + 클릭
  • 2. Add a suggestion 버튼 클릭
  • 3. 코드를 직접 수정하고 댓글 제출
  • 4. 저자가 Commit suggestion 버튼으로 한 번에 적용
```suggestion
<h2>Game over! Want to play again?!</h2>
```
실습 4단계에서 게임 메시지를 제안하고 Commit suggestion으로 바로 적용합니다.

PR 병합 (Merge)

리뷰와 피드백을 반영한 최종 통합

리뷰를 수집하고 피드백을 반영했으면, 이제 변경 사항을 main 브랜치에 병합할 차례입니다.

update-game ── ● ── ● ── suggestion ──┐
main ────────────────────── ● merge ← HEAD

병합 후 할 일

  • Merge pull request 클릭
  • Confirm merge 클릭
  • 사용한 브랜치 삭제 (선택 사항)
실습 5단계에서 Conversation 탭 하단의 Merge pull request를 클릭해 병합합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: PR 생성 → pull_request 이벤트 감지 → PR 구조 확인
  • 2단계: 담당자 지정 → assigned 이벤트 감지 → 본인 확인
  • 3단계: 리뷰 제출 → pull_request_review 감지 → 코멘트 확인
  • 4단계: 제안 적용 → push 이벤트 감지 → 커밋 확인
  • 5단계: PR 병합 → pull_request closed 감지 → merged 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
