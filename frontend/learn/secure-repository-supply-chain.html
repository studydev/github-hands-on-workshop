공급망 보안이란?

오픈소스 의존성이 만드는 보안 위험

대부분의 프로젝트는 수백 개의 오픈소스 의존성에 의존합니다. 이 의존성에 취약점이 있다면 공급망 공격의 위험에 노출됩니다.

GitHub의 공급망 보안 기능

  • 의존성 그래프 — 저장소의 모든 의존성 시각화
  • 의존성 검토 — PR에서 의존성 변경 확인
  • Dependabot 경고 — 취약점 알림
  • Dependabot 보안 업데이트 — 자동 수정 PR
  • Dependabot 버전 업데이트 — 최신 버전 유지
사용 중인 의존성의 취약점을 파악하고 패치하는 것이
공급망 보안의 핵심입니다.

의존성 그래프

저장소가 어떤 패키지에 의존하는지 한눈에 파악

의존성 그래프는 매니페스트 및 잠금 파일을 분석하여 저장소의 모든 의존성을 요약합니다.

보여주는 정보

  • Dependencies — 저장소가 의존하는 생태계와 패키지
  • Dependents — 해당 저장소에 의존하는 다른 저장소

실습에서 하는 일

  • Settings에서 Dependency Graph 활성화 확인
  • package-lock.json에 follow-redirects 의존성 추가
  • Insights → Dependency graph에서 확인
모든 공개 저장소에 기본 활성화 — 바로 사용할 수 있습니다!

Dependabot 경고

취약한 의존성을 자동으로 감지

Dependabot 경고는 코드가 안전하지 않은 패키지에 의존하고 있음을 알려줍니다. GitHub Advisory Database를 참조합니다.

경고 카테고리

  • GitHub 검토 권고 — GitHub가 검증한 보안 취약점
  • 미검토 권고 — 커뮤니티가 보고한 취약점

실습에서 하는 일

  • GitHub Advisory Database에서 follow-redirects 검색
  • Settings → Dependabot 경고 활성화
  • "Prototype Pollution in minimist" 경고에서 PR 생성
경고가 많다 = 의존성이 활발히 유지보수되고 있다는 신호!

Dependabot 보안 업데이트

취약점 발견 → 자동으로 수정 PR 생성

보안 업데이트가 활성화되면, Dependabot이 취약한 의존성을 감지하고 자동으로 PR을 열어 수정합니다.

수동 vs 자동

  • 수동 — 경고 확인 → PR 직접 생성 (2단계에서 경험)
  • 자동 — Dependabot이 직접 PR 생성 (3단계에서 활성화)

주의할 점

하나의 의존성(예: axios) 업데이트가 다른 전이적 의존성도 함께 해결할 수 있습니다. Dependabot은 불필요해진 PR을 자동으로 닫습니다.

한꺼번에 모든 PR을 머지하지 마세요!
Dependabot이 알아서 정리합니다.

Dependabot 버전 업데이트

최신 버전을 자동으로 유지하는 설정 파일

버전 업데이트는 dependabot.yml 설정 파일로 의존성을 항상 최신 릴리스로 유지합니다.

설정 예시

version: 2
updates:
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "monthly"
  - package-ecosystem: "nuget"
    directory: "/code/"
    schedule:
      interval: "weekly"
보안 업데이트는 취약점 수정,
버전 업데이트는 최신 버전 유지 — 목적이 다릅니다!

실습의 비밀 — 자동 검증

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

자동 검증 흐름

  • 1단계: package-lock.json에 의존성 추가 → push → 의존성 그래프 확인
  • 2단계: Dependabot 경고 활성화 → 경고 기반 PR → 머지
  • 3단계: 보안 업데이트 활성화 → 자동 PR 생성 → axios PR 머지
  • 4단계: dependabot.yml 커밋 → 버전 업데이트 트리거
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내를 남깁니다.
