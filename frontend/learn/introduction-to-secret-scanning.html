시크릿(Secret)이란?

서비스 인증에 사용되는 평문 자격 증명

시크릿은 서비스에 대한 접근을 인증하는 평문 문자열입니다. 매년 GitHub에서 백만 개 이상의 토큰이 발견됩니다.

대표적인 시크릿 유형

  • AWS — Access Key ID + Secret Access Key
  • Google — API Key, Service Account JSON
  • GitHub — Personal Access Token (PAT)
  • DB — MongoDB, PostgreSQL 연결 문자열

왜 위험한가?

  • 공격자가 시크릿으로 클라우드 리소스에 무단 접근
  • 커밋 히스토리에 영구 저장 — 파일 삭제만으로 부족
  • credentials.yml, .env 같은 파일이 주요 타깃
실습 1단계에서 비활성화된 예시 자격 증명을 커밋하여 시크릿 스캐닝의 동작을 확인합니다.

시크릿 스캐닝

저장소에 노출된 자격 증명을 자동으로 탐지

시크릿 보호(Secret Protection)는 평문 자격 증명을 식별하고, 제거하며, 처음부터 기록되지 않도록 규칙을 만드는 도구입니다.

동작 원리

  • 패턴 매칭 — 200개 이상의 서비스 제공자 패턴 지원
  • 커밋 스캔 — 새 커밋마다 자동으로 시크릿 탐지
  • 알림 생성 — 탐지 즉시 저장소 관리자에게 알림
  • 파트너 알림 — 일부 시크릿은 제공자에게 자동 전달

활성화 방법

Settings → Security → Advanced Security
→ Secret Protection: enabled
모든 플랜의 공개 저장소에서 무료로 사용할 수 있습니다.

알림 검토 & 분류

발견된 시크릿에 대한 체계적 대응 프로세스

시크릿이 감지되면 Security 탭에서 알림을 확인합니다. 동일 시크릿이 여러 위치에서 발견되어도 중복 알림은 생성되지 않습니다.

알림 분류 워크플로우

  • 필터링 — Provider 드롭다운으로 AWS, GitHub 등 필터
  • 상세 검토 — 알림 상태, 노출 시기, 감지 위치 확인
  • 조치 — 시크릿 공급자에서 즉시 비활성화(폐기)
  • 닫기 — Close as → Revoked + 감사 설명 기록
🚨 조치 없이 알림을 닫지 마세요 — 문제를 숨기고 잘못된 보안 감각을 줍니다.

푸시 보호 (Push Protection)

시크릿이 저장소에 기록되기 전에 차단

코드를 GitHub에 푸시할 때 시크릿 스캐닝이 높은 신뢰도의 시크릿을 검사하고, 감지된 경우 커밋을 차단합니다.

푸시 보호 흐름

코드 수정 ──→ git push ──→ 🛡️ 스캔 ──→ ❌ 차단

활성화 방법

Settings → Security → Advanced Security
→ Secret Protection: enabled
→ Push Protection: enabled
⚠️ 푸시 보호는 푸시 시점에만 스캔합니다. 로컬 커밋은 검사하지 않으므로, 시크릿이 깊이 있으면 히스토리에서 제거해야 합니다.

우회 & Advanced Security

테스트 시크릿 허용과 엔터프라이즈 보안 기능

테스트용 시크릿처럼 오탐(false positive)이 발생할 수 있습니다. 이때 푸시 보호를 우회할 수 있습니다.

우회 옵션

  • It's used in tests — 테스트에 사용되는 시크릿
  • It's a false positive — 시크릿이 아닌 경우
  • I'll fix it later — 나중에 수정 예정

GitHub Advanced Security 추가 기능

  • 사용자 정의 패턴 — 조직 고유의 시크릿 패턴 정의
  • 비파트너 패턴 — 비밀번호, RSA/SSH 키, DB 연결 문자열
  • CodeQL 코드 스캐닝 — 정적 분석으로 취약점 탐지
  • 보안 개요 — 조직 전체 보안 현황 대시보드

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: 시크릿 보호 활성화 → push 이벤트 감지 → 자격 증명 파일 확인
  • 2단계: 알림 검토 → issue_comment 감지 → 알림 해결 확인
  • 3단계: 푸시 보호 활성화 → push 이벤트 감지 → 우회 기록 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
