Codespaces란?

클라우드에서 호스팅되는 즉시 사용 가능한 개발 환경

Codespace는 저장소 설정 파일로 정의되는 클라우드 개발 환경입니다. Docker 컨테이너 위에서 실행되며, 로컬 설치 없이 브라우저만으로 개발할 수 있습니다.

주요 장점

  • 즉시 시작 — 저장소에서 바로 Codespace 생성
  • 브라우저 개발 — IDE 설치 없이 VS Code 사용
  • 사전 구성 — Features, 스크립트, 확장 프로그램 자동 설치
  • "내 PC에서는 되는데!" 문제 방지 — 일관된 환경
실습 1단계에서 저장소의 Code → Codespaces 탭에서 첫 Codespace를 생성합니다.

Dev Container 구성

.devcontainer/devcontainer.json으로 환경 정의

Codespace의 핵심은 .devcontainer/devcontainer.json 파일입니다. 이 파일로 Docker 이미지, 도구, 설정을 선언적으로 정의합니다.

주요 설정 항목

  • image — 기본 Docker 이미지 지정 (예: mcr.microsoft.com/devcontainers/base:debian)
  • name — 구성 식별 이름
  • features — 개발 도구 자동 설치
  • customizations — VS Code 확장 프로그램 설정
실습 2단계에서 .devcontainer/devcontainer.json을 만들고 커스텀 이미지를 지정합니다.

편집기 & 코드 푸시

Codespace에서 코드 수정하고 저장소에 반영하기

Codespace 안의 VS Code에서 파일을 편집하고, 터미널로 프로그램을 실행하고, Git으로 변경사항을 푸시합니다.

개발 워크플로우

  • 탐색기 — 왼쪽 사이드바에서 파일 탐색 · 편집
  • 터미널 — 하단 패널에서 명령 실행 (python, node 등)
  • 소스 제어 — Git add → commit → push
  • 원격 상태 — 좌측 하단에서 Codespace 연결 확인
실습 1단계에서 src/hello.py를 수정하고 git push로 변경사항을 저장소에 반영합니다.

Features · 확장 프로그램

devcontainer에 도구와 확장을 선언적으로 추가

Dev Container Features는 Python, Node 같은 도구를 JSON 한 줄로 설치하는 재사용 모듈입니다.

추가 방법

  • Features — "ghcr.io/devcontainers/features/python:1" 등을 features 블록에 추가
  • VS Code 확장 — customizations.vscode.extensions 배열에 확장 ID 추가
  • 명령 팔레트 — Codespaces: Add Dev Container Configuration Files 명령 활용
  • 우클릭 추가 — 확장 프로그램에서 Add to devcontainer.json
실습 3단계에서 Python Feature와 확장 프로그램을 devcontainer.json에 추가합니다.

생명주기 스크립트 & 리빌드

postCreateCommand로 환경 초기화 자동화하기

Dev Container 사양은 생명주기 스크립트를 제공하여 Codespace 생성 시 패키지 설치, 설정 등을 자동화합니다.

커스터마이즈 순서

  • 1. .devcontainer/postCreate.sh 스크립트 작성
  • 2. "postCreateCommand"를 devcontainer.json에 추가
  • 3. 변경사항을 커밋하고 푸시
  • 4. Codespaces: Rebuild Container 명령으로 리빌드
실습 3단계에서 postCreate.sh 스크립트를 만들고 Codespace를 리빌드하여 커스터마이징을 적용합니다.

최종 테스트 — 온보딩 검증

삭제하고 다시 만들어서 확인하기

Codespace의 진짜 테스트는 새 개발자의 입장이 되어 보는 것입니다. 기존 환경을 삭제하고 처음부터 다시 시작해 봅니다.

테스트 워크플로우

  • 1. 기존 Codespace 삭제 (Code 버튼 → ⋯ → Delete)
  • 2. 새 Codespace 생성 (+ 또는 Create codespace on main)
  • 3. 확장 프로그램, Python 버전, 커스텀 스크립트 확인
  • 4. github.com/codespaces에서 전체 관리 가능
실습 4단계에서 기존 Codespace를 삭제하고, 새로 생성하여 모든 설정이 적용되었는지 검증합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: 코드 푸시 → push 이벤트 감지 → hello.py 변경 확인
  • 2단계: devcontainer.json 커밋 → push 이벤트 감지 → 구성 파일 확인
  • 3단계: Features/확장 추가 → push 이벤트 감지 → 커밋 내용 확인
  • 4단계: 이슈 댓글 → issue_comment 감지 → 완료 확인
