workflow_call 트리거

워크플로우를 재사용 가능하게 만드는 마법의 키워드

재사용 가능한 워크플로우는 workflow_call 트리거를 포함하는 워크플로우입니다. 다른 워크플로우에서 마치 함수처럼 호출할 수 있습니다.

같은 CI 로직을 10개 저장소에 복사? No!
한 곳에서 정의하고 어디서나 호출하세요.

기본 구조

name: Reusable Node Quality
on:
  workflow_call:
    inputs:
      node-version:
        description: "Node.js version"
        required: false
        default: "20"
        type: string
permissions:
  contents: read

입력 & 출력 (Inputs / Outputs)

재사용 워크플로우를 유연하게 만드는 파라미터

재사용 가능한 워크플로우는 inputs로 데이터를 받고, outputs로 결과를 반환하여 다양한 프로젝트에 맞게 조정됩니다.

입력 타입

  • string — 문자열 (예: Node 버전)
  • number — 숫자
  • boolean — true / false

출력 예시

outputs:
  page_url:
    description: "Deployed URL"
    value: ${{ jobs.deploy.outputs.url }}
입력은 ${{ inputs.node-version }}으로,
출력은 needs.<job>.outputs.<key>로 참조합니다.

호출자 워크플로우 (Caller)

재사용 가능한 워크플로우를 호출하는 방법

재사용 가능한 워크플로우를 uses: 키워드로 호출합니다. 같은 저장소 또는 다른 저장소의 워크플로우를 참조할 수 있습니다.

참조 형식

  • 같은 저장소 — ./.github/workflows/reusable.yml
  • 다른 저장소 — owner/repo/.github/workflows/wf.yml@v1
jobs:
  quality:
    uses: ./.github/workflows/reusable-node-quality.yml
    with:
      node-version: 24
다른 저장소의 워크플로우를 사용할 때는
안정적인 태그나 커밋 SHA에 고정하세요.

권한 상속 (Permissions)

호출자 → 재사용 워크플로우 간 토큰 권한 흐름

재사용 워크플로우를 호출하면 토큰 권한이 상속되며, 동일하거나 더 제한적으로만 변경됩니다.

권한 규칙

  • 호출자 — 전체 권한의 상한을 설정
  • 재사용 워크플로우 — 동일하거나 더 좁게만
  • 중첩 호출 — 직접 부모로부터 상속
jobs:
  github-pages:
    needs: quality
    uses: ./.github/workflows/deploy-pages.yml
    permissions:
      contents: read
      pages: write
      id-token: write
권한은 확대 불가 — 호출자 수준에서
설정한 상한을 넘을 수 없습니다.

중첩 워크플로우 & Pages 배포

워크플로우를 조합하여 완전한 CI/CD 파이프라인 구축

재사용 가능한 워크플로우는 중첩될 수 있습니다. 최대 4단계 깊이까지 호출 가능하며, needs:로 의존 관계를 설정합니다.

CI 파이프라인 예시

  • quality — 린트 + 테스트 + E2E (재사용)
  • github-pages — Pages 배포 (재사용, needs: quality)
  • comment — PR에 배포 URL 댓글
comment:
  if: always()
  needs: github-pages
  runs-on: ubuntu-latest
  steps:
    - uses: GrantBirki/comment@v2.1.1
      with:
        body: |
          🌐 ${{ needs.github-pages
                  .outputs.page_url }}
if: always()로 배포 실패 시에도
PR 댓글에서 로그 링크를 확인할 수 있습니다.

실습의 비밀 — 자동 검증

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

자동 검증 흐름

  • 1단계: reusable-node-quality.yml 생성 → push → YAML 구조 확인
  • 2단계: ci.yml 생성 & PR 오픈 → pull_request → 재사용 호출 확인
  • 3단계: Pages 배포 & 댓글 → push → 배포 + 권한 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내를 남깁니다.
