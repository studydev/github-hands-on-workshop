Model Context Protocol (MCP)

AI를 위한 USB-C — 범용 커넥터

MCP는 AI 도구가 외부 서비스와 원활하게 상호작용할 수 있게 하는 표준 프로토콜입니다.

서비스의 기능과 요구 사항을 스키마로 설명하면, AI가 어떤 메서드를 호출할지 결정하고 매개변수를 정확히 제공합니다.

하나의 MCP 서버를 연결하면 GitHub, Slack, Figma 등
수십 개의 도구를 Copilot 에이전트에 장착할 수 있습니다.

MCP 서버 설정 방법

  • .vscode/mcp.json 파일 생성
  • 서버 URL 등록: https://api.githubcopilot.com/mcp/
  • Start 클릭 → GitHub 인증 → 도구 카탈로그 로드

에이전트 모드 — MCP 도구 호출

Copilot이 스스로 계획하고 도구를 호출한다

프롬프트를 보내면 Copilot은 사용 가능한 도구 카탈로그(목록 + 스키마)를 함께 분석합니다.

동작 흐름

  • 이 요청에 도구가 필요한가? → 판단
  • 어떤 도구가 의도에 가장 적합한가? → 선택
  • 입력 스키마에 따라 인수를 구성 → 호출
  • 결과를 LLM에 스트리밍 → 응답 생성
#create_pull_request, #codebase 등
#도구이름으로 특정 도구를 명시적으로 지정할 수 있습니다.

GitHub MCP로 가능한 것

  • 유사 공개 프로젝트 검색 & 비교
  • 이슈 조회·생성·댓글
  • 브랜치 생성 → 코드 변경 → 풀 리퀘스트

아이디어 탐색 & 이슈 생성

Step 2 — 조사하고, 비교하고, 즉시 이슈로 저장

GitHub MCP 서버를 활용하면 Copilot이 직접 유사 프로젝트를 검색하고, 기능을 비교한 뒤, 새 아이디어를 이슈로 생성합니다.

실습 프롬프트 예시

  • "과외 활동 관리 프로젝트 검색해줘"
  • "3번째 프로젝트의 기능을 자세히 설명해줘"
  • "우리 프로젝트와 비교해서 새로운 기능은?"
  • "마음에 들어! 이슈로 만들어줘"
MCP 도구 호출 시 Copilot이 권한을 요청합니다.
요청 내용을 확인한 후 Continue를 클릭하세요.

이슈에서 PR까지 한 번에

Step 3 — 에이전트가 분류·구현·PR 생성

Copilot 에이전트 모드에서 열린 이슈를 분석하고, 중요도를 판단한 뒤, 코드 변경과 PR 생성까지 수행합니다.

워크플로우

  • 이슈 탐색 — "열린 이슈가 몇 개야?"
  • 우선순위 판단 — "가장 중요한 3개를 골라줘"
  • 구현 — "첫 번째를 구현해줘: 새 브랜치 → 변경 → PR"
⚠️ Copilot이 수행하려는 작업을 항상 확인하세요.
MCP 서버의 외부 기능은 실행 취소가 불가능할 수 있습니다.

도구 사전 승인

도구를 자주 사용하면 나머지 대화 세션 동안 사전 권한 부여가 가능합니다.

AI 생성 코드 검증하기

Step 4 — 리뷰, 머지, 그리고 감사 댓글

AI 어시스턴트가 생산성을 높여주지만, 생성 여부에 관계없이 모든 작업을 검토하고 검증하는 것은 개발자의 책임입니다.

검증 체크리스트

  • PR의 변경사항 (diff) 검토
  • Copilot 코드 리뷰 기능 활용 (Pro 구독)
  • 만족스러우면 머지
  • 완료된 이슈에 감사 댓글 추가
실제 프로젝트에서는 GitHub Actions로
자동화된 테스트를 설정하여 AI 코드를 검증합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: mcp.json 커밋 → push 이벤트 → MCP 설정 확인
  • 2단계: 이슈 생성 → issues 이벤트 → 이슈 존재 확인
  • 3단계: PR 생성 → pull_request 이벤트 → PR 구조 확인
  • 4단계: 이슈 댓글 → issue_comment 이벤트 → 댓글 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
