CodeQL 언어 매트릭스란?

단일 워크플로우로 여러 언어를 동시에 스캔

언어 매트릭스는 하나의 CodeQL 워크플로우 파일에서 여러 프로그래밍 언어를 관리하는 GitHub Actions 전략입니다.

매트릭스 사용의 이점

  • 간결성 — 각 언어별 별도 워크플로우 불필요, 단일 파일로 관리
  • 유연성 — 언어 추가/제거가 매트릭스 배열 수정만으로 가능
  • 일관성 — 모든 언어에 동일한 스캐닝 정책 자동 적용

지원 언어

c-cpp · csharp · go · java-kotlin
javascript-typescript · python · ruby · swift
잘 구성된 CodeQL 설정은 안전하고 신뢰할 수 있는 코드베이스를 유지하는 핵심입니다.

Strategy 매트릭스 구성

codeql.yml의 strategy 섹션에 언어 배열 정의

CodeQL 워크플로우의 jobs 섹션에 strategy.matrix를 추가하여 스캔할 언어를 정의합니다.

YAML 구성

strategy:
  fail-fast: false
  matrix:
    language:
      - 'go'
      - 'java-kotlin'
      - 'javascript-typescript'
      - 'python'

핵심 포인트

  • fail-fast: false — 하나의 언어 실패가 다른 언어 스캔을 중단시키지 않음
  • language 배열 — 프로젝트에서 사용하는 언어만 선택
실습 1단계에서 steps 섹션 위에 strategy 블록을 추가합니다.

Init & Analyze에 매트릭스 적용

CodeQL 액션에 matrix.language 변수 연결

매트릭스를 정의한 후, CodeQL init과 analyze 액션에 언어 변수를 전달해야 합니다.

Initialize CodeQL

- uses: github/codeql-action/init@v3
  with:
    languages: ${{ matrix.language }}

Perform CodeQL Analysis

- uses: github/codeql-action/analyze@v3
  with:
    category: ${{ matrix.language }}
  • languages — 해당 매트릭스 항목의 언어로 CodeQL DB 초기화
  • category — 분석 결과를 언어별로 구분하여 저장
매트릭스의 각 언어에 대해 별도의 잡이 병렬로 실행되어 스캔 속도가 향상됩니다.

CodeQL Autobuild

컴파일 언어를 자동으로 빌드하여 분석 가능하게

Autobuild는 저장소의 빌드 시스템을 자동 감지하고 적절한 명령을 실행하여 컴파일 언어를 빌드합니다.

왜 필요한가?

  • 컴파일 언어(Java, C/C++ 등)는 분석 전에 빌드가 필요
  • 인터프리터 언어(Python, JS 등)는 빌드 없이 바로 분석 가능
  • Autobuild가 빌드 시스템을 감지하여 자동 컴파일

Autobuild 단계 추가

- name: Autobuild
  uses: github/codeql-action/autobuild@v3
Initialize CodeQL과 Perform CodeQL Analysis 사이에 autobuild 단계를 배치합니다.

조건부 실행 (if 표현식)

컴파일 언어에만 autobuild를 실행하는 조건 분기

모든 언어에 autobuild를 실행하면 불필요한 시간이 소모됩니다. 조건부 표현식으로 필요한 언어에만 적용합니다.

조건부 Autobuild

- if: ${{ contains(matrix.language, 'java-kotlin') }}
  name: Autobuild
  uses: github/codeql-action/autobuild@v3

동작 방식

  • contains() — 매트릭스 언어에 'java-kotlin'이 포함되는지 확인
  • 조건이 true인 잡에서만 autobuild 단계가 실행됨
  • Python, JS 등 인터프리터 언어 잡에서는 건너뜀
실습 2단계에서 Autobuild 단계에 if 조건을 추가하여 java-kotlin에만 실행되도록 합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 보안 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: codeql.yml에 strategy.matrix 추가 → 매트릭스 구성 검증
  • 2단계: autobuild + if 조건 추가 → 조건부 실행 검증

최종 워크플로우 구조

strategy → matrix → language[]
↓ Initialize CodeQL (languages: matrix.language)
↓ Autobuild (if: java-kotlin)
↓ Perform CodeQL Analysis (category: matrix.language)
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
