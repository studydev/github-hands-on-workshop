GitHub 코드 스캐닝

개발 프로세스에 직접 통합되는 보안 테스트

코드 스캐닝은 GitHub Advanced Security (GHAS) 제품군의 일부로, SAST·컨테이너·IaC 등을 지원합니다.

주요 특징

  • 결과가 코드 바로 옆에 — 컨텍스트 전환 없이 GitHub에서 확인
  • 공개 저장소 무료 — 비공개는 GHAS 유료 라이선스 필요
  • PR 통합 — 머지 전에 취약점을 잡아냄

설정 방법

Settings → Security → Advanced Security
→ Code scanning → CodeQL → Set up → Default
→ Enable CodeQL 클릭
실습 1단계에서 Settings → Code scanning에서 CodeQL 기본 설정을 활성화합니다.

CodeQL 분석 엔진

코드를 데이터처럼 쿼리하는 정적 분석 도구

CodeQL은 코드를 데이터베이스로 변환한 뒤, SQL과 유사한 쿼리 언어로 보안 패턴을 검색하는 정적 분석 도구입니다.

탐지 가능한 취약점

  • SQL 인젝션 — 사용자 입력이 쿼리에 직접 삽입
  • 크로스 사이트 스크립팅 (XSS) — 악성 스크립트 주입
  • 코드 인젝션 — 외부 입력으로 임의 코드 실행

쿼리 스위트

Default — 일반적인 보안 패턴
Extended — 더 많은 패턴 포함
Custom — GitHub Actions로 직접 구성
보안 전문가 팀이 사전 구성한 스위트를 활용하면 별도 설정 없이도 강력한 분석이 가능합니다.

PR에서 취약점 탐지

머지 전에 보안 문제를 잡아내는 자동 검사

Pull Request를 생성하면 CodeQL이 자동으로 코드 변경사항을 분석합니다.

SQL 인젝션 예시

- "SELECT * FROM books WHERE name LIKE '%s'" % name
+ "SELECT * FROM books WHERE name LIKE '%" + name + "%'"
⚠ 사용자 입력이 쿼리에 직접 삽입 → SQL 인젝션!

PR 체크 결과

  • PR 하단 Checks에서 CodeQL 체크 확인
  • 취약점 발견 시 댓글로 알림 + 수정 제안
  • Show paths 링크로 소스→싱크 흐름 추적
실습 2단계에서 routes.py에 SQL 인젝션 취약점을 도입하고 PR 체크 결과를 확인합니다.

알림 검토 및 분류

Security 탭에서 알림 관리 — 해제, 재오픈, 감사 추적

머지 후 Security → Code scanning에서 열린 알림을 확인하고 관리합니다.

알림 정보 구성

  • 해결 상태 — Open / Dismissed / Fixed
  • 영향받는 브랜치 — main, feature 등
  • 심각도 — Critical, High, Medium, Low
  • CWE — 소프트웨어 약점 분류 체계
  • CVE — 공개 취약점 식별 번호

분류 워크플로우

알림 확인 → Dismiss alert (사유 선택)
→ 설명 입력 → 감사 추적에 기록
→ 필요 시 Reopen alert로 재오픈
실습 3단계에서 알림을 해제(Dismiss)하고 다시 열어(Reopen) 감사 추적을 확인합니다.

보안 취약점 수정

파라미터화된 쿼리로 SQL 인젝션 방지

CodeQL이 제안하는 수정 방법을 참고하여 코드를 패치합니다.

수정 전 vs 수정 후

- "SELECT * FROM books WHERE name LIKE '%" + name + "%'"
+ "SELECT * FROM books WHERE name LIKE %s", name
✓ 파라미터 바인딩으로 인젝션 방지

수정 후 확인

  • main 브랜치에 직접 커밋 → CodeQL 재스캔
  • Security 탭에서 열린 알림 0개 확인
  • 닫힌 알림의 감사 추적에서 자동 수정 기록 확인
실습 4단계에서 routes.py를 수정하고 CodeQL 재스캔으로 알림이 해결되었는지 확인합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 보안 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: CodeQL 활성화 → 설정 변경 감지 → 스캐닝 시작
  • 2단계: 취약 코드 PR → CodeQL 체크 → 알림 생성
  • 3단계: 알림 분류 → Dismiss/Reopen → 감사 기록
  • 4단계: 코드 수정 → 재스캔 → 알림 자동 해결
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
