버전 관리 (Version Control)

코드 변경의 타임머신 — 분산형 버전 관리 시스템

Git은 분산 버전 관리 시스템으로, 각 개발자가 프로젝트 히스토리의 완전한 복사본을 갖고 있습니다.

이런 경험이 있다면?

  • my-project-final-v2-really3.zip
  • "언제 작동이 멈춘 거지? 아무것도 안 바꿨는데!"
  • "파일이 잠겨 있어요. 월요일까지 복사본으로 작업할게요."

Git의 장점

  • 오프라인 작업 — 대부분의 작업이 로컬에서 처리
  • 향상된 안정성 — 분산 복사본이 백업 역할
  • 유연한 워크플로우 — 자신만의 프로세스와 도구 사용 가능
실습 1단계에서 Codespace 환경에서 git --version으로 Git을 확인하고 사용자 정보를 설정합니다.

Git 워크플로우

세 영역을 오가는 변경 관리의 흐름

Git의 핵심은 세 가지 영역으로 구성됩니다:

Working Directory ──→ Staging Area ──→ Repository
파일 수정                 git add              git commit

각 영역의 역할

  • 작업 디렉토리 — 변경 사항을 만드는 프로젝트 파일
  • 스테이징 영역 — 커밋할 변경 사항을 그룹화하는 준비 영역
  • 저장소 — 프로젝트 히스토리의 영구 기록
git add로 스테이징, git commit으로 히스토리에 기록합니다.

커밋 (Commit)

변경 사항을 영구 기록하는 스냅샷

커밋은 프로젝트의 특정 시점 스냅샷입니다. 커밋 메시지와 함께 히스토리에 영구 저장됩니다.

핵심 명령어

git init — 새 저장소 시작
git add src/* — 파일을 스테이징
git commit -m "Initial commit" — 커밋
git status — 현재 상태 확인
main ── ● Initial commit ── ● Start docs ── HEAD
실습 2단계에서 게임 파일을 git init으로 초기화하고 첫 커밋을 합니다.

히스토리 (History)

커밋 체인으로 프로젝트의 과거를 탐색

Git은 커밋을 통해 프로젝트의 완전한 히스토리를 유지합니다. 각 커밋에는 고유 해시 ID, 작성자, 타임스탬프, 메시지가 포함됩니다.

핵심 명령어

git log — 상세한 히스토리 표시
git log --oneline — 한 줄씩 간결하게
git log --graph --oneline — 시각적 그래프
git checkout <commit-id> — 과거 버전으로 이동

HEAD 포인터는 현재 위치를 나타내는 특별한 라벨입니다.

실습 3단계에서 git log로 히스토리를 탐색하고 git checkout으로 과거 버전을 확인합니다.

Diff — 변경 비교

파일의 이전 vs 이후를 한눈에 확인

Git은 기호와 색상으로 파일 변경 사항을 보여줍니다:

+ 이것은 추가된 줄입니다 (녹색)
- 이것은 삭제된 줄입니다 (빨간색)

핵심 명령어

git diff — 작업 디렉토리 vs 스테이징
git diff --staged — 스테이징 vs 마지막 커밋
git diff HEAD~1 — 현재 vs 이전 커밋
실습 4단계에서 게임 코드를 수정하고 git diff로 변경 사항을 확인합니다. VS Code의 Diff 뷰도 실습합니다.

브랜치 & 머지

안전한 병렬 작업 공간에서 기능 개발 후 통합

브랜치는 특정 커밋을 가리키는 가벼운 포인터입니다. 원본에 영향 없이 병렬로 작업할 수 있습니다.

main ── ● ── ● ──────────── ● merge ← HEAD
                 └── feature ── ●── ● ──┘

핵심 명령어

git branch feature — 새 브랜치 생성
git checkout feature — 브랜치 전환
git merge feature — 브랜치 병합
git branch --delete feature — 브랜치 삭제
실습 5단계에서 기능 브랜치를 만들고 게임에 최고 점수, 레벨 기능을 추가한 뒤 머지합니다.

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: 사용자 설정 → git config 감지 → 설정 확인
  • 2단계: 저장소 초기화 → push 이벤트 감지 → 커밋 확인
  • 3단계: 히스토리 탐색 → git log 실행 확인
  • 4단계: Diff 비교 → push 이벤트 감지 → 변경 확인
  • 5단계: 브랜치 머지 → merge 이벤트 감지 → 통합 확인
  • 6단계: 협업 개념 복습 → 퀴즈 응답 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
