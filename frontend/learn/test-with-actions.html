지속적 통합 (CI)

커밋할 때마다 자동으로 품질을 확인

CI(Continuous Integration)는 코드 변경을 저장소에 통합할 때마다 자동으로 빌드와 테스트를 실행하는 개발 방식입니다.

목표: main 브랜치에 깨진 코드가 들어가지 않도록 방지하는 것!

CI의 핵심 원칙

  • 자동 테스트 — 커밋마다 테스트 자동 실행
  • 빠른 피드백 — PR에 결과 즉시 표시
  • 공유 책임 — 팀 전체가 품질 유지
  • 작은 변경 — 자주, 조금씩 통합

이 실습에서는 Python pytest와 커버리지를 GitHub Actions로 자동화합니다.

워크플로우 구성 요소

워크플로우 → 잡 → 스텝 → 액션

GitHub Actions 워크플로우는 계층 구조로 구성됩니다.

  • 워크플로우 — 전체 자동화 단위 (.yml 파일)
  • 잡 (Job) — 격리된 환경에서 실행되는 작업 그룹
  • 스텝 (Step) — 잡 안의 개별 명령 (run: 또는 uses:)
  • 액션 (Action) — 재사용 가능한 자동화 조각
name: Python Package
on:
  pull_request:
    branches: ["main"]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pytest --verbose

테스트 자동화 (pytest)

Python의 강력한 테스트 프레임워크

pytest는 Python에서 가장 인기 있는 테스트 프레임워크입니다. 간결한 문법으로 단위 테스트를 작성할 수 있습니다.

테스트 작성 패턴

  • Arrange — 테스트 데이터 준비
  • Act — 함수 실행
  • Assert — 결과 검증
def test_get_nth_fibonacci_ten():
    # Arrange
    n = 10
    # Act
    result = get_nth_fibonacci(n)
    # Assert
    assert result == 55
pytest --verbose로 상세한 테스트 결과를 확인하세요!

커버리지 리포트

코드의 몇 %가 테스트되고 있는가?

코드 커버리지는 테스트가 실행하는 코드의 비율을 측정합니다. coverage + pytest-cov로 리포트를 생성합니다.

워크플로우에서 커버리지 활용

  • pytest --cov=src로 커버리지 측정
  • PR 댓글에 커버리지 리포트 자동 게시
  • 최소 기준 설정 (예: 90% 이상)
  • 기준 미달 시 워크플로우 실패 처리
- name: Run tests
  run: pytest --cov=src
- name: Coverage comment
  uses: py-cov-action/python-coverage-
        comment-action@v3
- name: Fail if below threshold
  run: coverage report --fail-under=90

브랜치 보호 & Rulesets

테스트 없이는 병합할 수 없다!

워크플로우가 실행되더라도 테스트 실패를 무시하고 병합할 수 있다면 의미가 없습니다. Rulesets으로 이를 방지합니다.

Ruleset 설정 방법

  • Settings → Rules → Rulesets으로 이동
  • 대상 브랜치: main
  • Require status checks to pass 활성화
  • 필수 체크: python-coverage 워크플로우
테스트 실패 시 Merge 버튼이 비활성화됩니다!
깨진 코드가 프로덕션에 도달하는 것을 차단합니다.

실습의 비밀 — 자동 검증

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

자동 검증 흐름

  • 1단계: Codespace에서 테스트 실행 → 이슈 댓글 → Mona 확인
  • 2단계: 워크플로우 파일 추가 → push 이벤트 → YAML 구조 확인
  • 3단계: PR 생성 → pull_request → 워크플로우 실행 확인
  • 4단계: 테스트 수정 & 커버리지 90% → 병합 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내를 남깁니다.
