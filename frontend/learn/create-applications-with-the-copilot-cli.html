Copilot CLI란?

GitHub Copilot의 독립 실행형 터미널 애플리케이션

GitHub Copilot CLI는 Copilot의 기능을 커맨드 라인으로 직접 가져오는 npm 패키지입니다. OpenAI와 Google의 최신 AI 모델로 구동됩니다.

$ npm install -g @github/copilot
설치 후 copilot 명령어로 세션을 시작합니다.

에디터 vs CLI

  • 인터페이스 — VS Code 대신 터미널에서 직접 작업
  • 대화형 — 자연어로 코드 생성·수정·테스트
  • 이미지 지원 — 붙여넣기·드래그 앤 드롭으로 시각적 컨텍스트
  • MCP 통합 — --enable-all-github-mcp-tools
  • 헤드리스 모드 — -p 옵션으로 자동화 지원

설치 요구 사항

Node.js 22+ + npm 10+ + Copilot 구독

주요 기능과 명령어

터미널에서 사용하는 슬래시 명령어와 단축키

슬래시 명령어

  • /session — 현재 채팅 세션 세부 정보 보기
  • /context — 토큰 사용량 시각적 개요
  • /usage — 프리미엄 요청 수·세션 시간·토큰 분석
  • /share [file|gist] — 세션을 마크다운이나 gist로 저장
  • /delegate — 코딩 에이전트에 작업 위임
  • /agent <name> — 커스텀 에이전트 호출

전역 단축키

  • @ — 파일 멘션하여 컨텍스트에 포함
  • ! — 셸 명령 직접 실행 (Copilot 우회)
  • Shift+Tab — 플랜 모드 ↔ 대화형 모드 전환
  • Ctrl+O — 실행 결과 상세 확인

대화형 개발

CLI에서 자연어로 코드 생성부터 테스트까지

Copilot CLI는 터미널에서 풍부한 대화형 경험을 제공합니다. 코드에 대해 자연스러운 대화를 나누고, 이미지를 기반으로 코드를 생성할 수 있습니다.

대화형 워크플로우

프롬프트 입력 → 코드 생성 → 실행 & 테스트 → 반복 개선

이미지로 코드 생성

  • 파일 참조 — @images/screenshot.png으로 이미지 컨텍스트 제공
  • 드래그 & 드롭 — 터미널에 이미지 직접 끌어다 놓기
  • 이슈 연동 — GitHub 이슈 내용을 참고해 구현

커스텀 에이전트

  • .github/agents/에 에이전트 프로필 생성
  • 문서화·보안·인프라 등 도메인별 특화 워크플로우
  • /agent <name>으로 호출

/delegate — 작업 위임

더 큰 작업은 코딩 에이전트에게 맡기기

/delegate 명령어를 사용하면 Copilot 코딩 에이전트에 작업을 할당하여 백그라운드에서 자율적으로 수행할 수 있습니다.

위임 워크플로우

/delegate 명령 → 브랜치 생성 → 자율적 구현 → 드래프트 PR

동작 과정

  • 자동 브랜치 생성 — 코딩 에이전트가 새 브랜치를 만듦
  • 드래프트 PR — 자동으로 풀 리퀘스트 열기
  • 자율 작업 — 백그라운드에서 구현 수행
  • 실시간 스트리밍 — 출력이 터미널로 전달
  • 리뷰 요청 — 완료 시 검토 알림
/delegate는 Copilot 코딩 에이전트(CCA)를 활용하므로
프리미엄 요청을 소비합니다. 일반 CLI 사용은 무료 모델도 가능!

반복적 개발

개발 모멘텀을 유지하며 빠르게 확장하기

Copilot CLI는 터미널에 머물면서 컨텍스트 전환 없이 개발 모멘텀을 유지하도록 도와줍니다.

반복 개발 사이클

  • 이슈 생성 — CLI에서 직접 GitHub 이슈 만들기
  • 코드 생성 — 최신 AI 모델로 빠르게 구현
  • 테스트 작성 — 에지 케이스 포함한 포괄적 테스트
  • 디버깅 — 문제 진단 & 수정
  • 커밋 & 푸시 — git 작업까지 CLI에서 처리

생산성 팁

/share gist로 세션을 저장하여 나중에 참고하세요.
-p 헤드리스 모드로 반복 작업을 자동화할 수 있습니다.
  • 장기 실행 명령을 효율적으로 처리
  • 모범 사례와 디자인 패턴 제안
  • 코드 리팩토링 & 문서 생성 지원

PR과 코드 리뷰

CLI에서 PR 생성부터 병합까지 전체 워크플로우

풀 리퀘스트 이해하기

  • 병합 전에 팀원이 코드를 리뷰할 수 있음
  • 변경 사항과 결정의 히스토리 유지
  • 이슈에 연결하여 자동 닫기 가능
  • CI/CD를 통한 자동 검증

AI 지원 코드 리뷰

  • Copilot을 리뷰어로 추가하여 피드백 받기
  • 코드 품질 개선·버그 식별·패턴 추천
  • CLI를 떠나지 않고 워크플로우 전체 자동화

병합 워크플로우

PR 생성 → 이슈 연결 → Copilot 리뷰 → 스쿼시 병합

GitHub Actions — 실습의 비밀

뒤에서 자동으로 돌아가는 검증 시스템

이 실습은 GitHub Actions를 사용해 여러분의 진행을 자동으로 확인합니다.

동작 원리

  • 1단계: CLI 설치 → 이슈 생성 감지 → "calculator" 키워드 확인
  • 2단계: 코드 생성 → push 이벤트 감지 → calculator.js & 테스트 확인
  • 3단계: 기능 확장 → push 이벤트 감지 → 새 연산 & 테스트 확인
  • 4단계: PR 병합 → pull_request closed 감지 → 이슈 닫힘 확인
각 단계를 완료하면 Mona(봇)가 자동으로 다음 단계 안내 코멘트를 남깁니다.
